How it works

1

Reads your sessions

Parses ~/.claude/projects/**/*.jsonl — the session logs Claude Code already writes.

2

Detects patterns

Finds recurring bash commands and tool-call sequences across sessions. Extracts intent from the assistant's reasoning chain — not just what ran, but why.

3

Generates SKILL.md

Writes a skill file per pattern, ready to drop into Claude Code, Cursor, Copilot, Gemini CLI, or any agent that reads SKILL.md.

Intent extraction

Most tools see what you ran. SkillMind reads the assistant's reasoning in the turns before each command and stitches them into intent context — so generated skills explain when to use a pattern, not just how.

Quickstart

# clone and install
$ git clone https://github.com/gocoolp/skillmind
$ cd skillmind && uv sync

# see what patterns exist in your sessions
$ uv run skillmind scan
✓ 48 sessions · 1,204 tool calls

Top bash command patterns
git status    ████████   42×  12 sessions
gh pr create  ██████     28×   9 sessions
pytest        █████      21×   7 sessions

# generate SKILL.md files
$ uv run skillmind suggest --output ./skills --top-n 5
No API key? SkillMind generates skeleton files you can fill in manually. Set ANTHROPIC_API_KEY for AI-drafted content.

LLM backends

Set any key and SkillMind uses it. Fall through to skeleton mode with no key — fully offline, no data sent anywhere.

Env varBackend
ANTHROPIC_API_KEYClaude (recommended)
OPENAI_API_KEY + OPENAI_BASE_URLOpenAI, Azure, or any compatible endpoint
SKILLMIND_OLLAMALocal Ollama — fully air-gapped
(none)Skeleton mode — offline, no API needed

Everything stays local

SkillMind reads logs on your machine and writes files on your machine. Session data never leaves your environment. When you set an API key, only the extracted pattern summary and intent snippets are sent — never raw session content.
